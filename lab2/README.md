# Система управления библиотекой

Система автоматизации учета библиотечного фонда, реализованная на Python с использованием графического фреймворка PySide6. Проект построен на строгом соблюдении архитектурного паттерна MVC, принципов ООП, стандартов PEP8 и строгой типизации.

## Архитектура проекта

Проект разделен на три основных слоя в соответствии с паттерном Model-View-Controller: данные и бизнес-логика (model), графический интерфейс (view) и управление (controller).

---

### Основные сущности

* книга - дата-класс `Book`
* менеджер базы данных - `DatabaseHandler`
* обработчики XML - `XmlHandler`, `XmlLoader`
* главное окно приложения - `MainWindow`
* модель данных таблицы - `BookTableModel`
* контроллер приложения - `MainController`

### Реализованные поведения
* операция добавления новых книг
* операция многокритериального поиска книг
* операция удаления записей по заданным фильтрам
* операция пагинации (постраничного вывода) данных
* операция переключения форматов отображения (Таблица / Иерархическое дерево)
* операция импорта и экспорта базы данных в формат XML

### Описание классов

**MainController**: Центральный узел системы (Слой Controller). Выступает медиатором между пользовательским интерфейсом и базой данных. Управляет состоянием пагинации (текущая страница, лимиты, смещения) и координирует передачу данных.

#### Слой данных (Model)
* **Book**: Сущность данных (dataclass), представляющая книгу. Содержит атрибуты (название, автор, издатель, количество томов, тираж) и автоматически высчитывает общее количество книг (`total_volumes`).
* **DatabaseHandler**: Инкапсулирует логику взаимодействия с реляционной базой данных SQLite. Реализует подготовленные SQL-запросы для безопасного добавления, поиска, удаления и получения записей с учетом `LIMIT` и `OFFSET`.
* **XmlHandler / XmlLoader**: Отвечают за сериализацию и десериализацию данных. `XmlHandler` использует DOM-подобный подход (`minidom`) для генерации иерархии при экспорте. `XmlLoader` реализует событийно-ориентированный потоковый SAX-парсер (`BookSaxHandler`) для эффективного импорта XML без перегрузки оперативной памяти.

#### Слой представления (View)
* **MainWindow**: Главное окно приложения (наследует `QMainWindow`). Агрегирует меню, панели инструментов и систему пагинации. Использует `QStackedWidget` для переключения между табличным и древовидным представлениями.
* **BookTableModel**: Наследник `QAbstractTableModel`. Прослойка (адаптер) между сырыми данными `List[Book]` и компонентом `QTableView`, обеспечивающая быструю отрисовку только видимых ячеек.
* **AddBookDialog / SearchDialog / DeleteDialog**: Модальные окна (`QDialog`), предоставляющие формы ввода для взаимодействия с пользователем и возвращающие валидированные данные контроллеру.

---

Пользователь системы может:
* просматривать каталог книг с использованием постраничной навигации
* менять количество отображаемых записей на одной странице
* переключать вид каталога между таблицей и деревом свойств
* добавлять новые издания, заполняя необходимые поля
* искать и удалять книги по сложным составным критериям (например: конкретный автор + тираж больше заданного значения)
* сохранять текущую базу данных в XML-файл или загружать данные из стороннего XML-документа с автоматической очисткой старой базы

### Интерфейс (Сценарии демонстрации)

Система предлагает полноценный графический интерфейс (GUI). Ниже описаны основные сценарии взаимодействия:

**Сценарий 1: Пагинация и настройка вида**
1. При запуске открывается главное окно с загруженной первой страницей записей (по умолчанию 10 записей на страницу).
2. На нижней панели отображается статус: `Страница: 1/5 | Всего записей: 50`.
3. Пользователь вводит `20` в поле `Записей на стр:` (QSpinBox). Контроллер автоматически пересчитывает лимиты, запрашивает новые данные у БД и обновляет таблицу. Статус меняется на: `Страница: 1/3 | Всего записей: 50`.
4. В верхнем меню `Вид` пользователь нажимает `Таблица/Дерево`. Таблица скрывается, и появляется `QTreeView`, где каждая книга является корневым узлом, развернув который можно увидеть детали (автор, издательство, тираж).

**Сценарий 2: Добавление новой книги**
1. В верхнем меню выбирается `Записи -> Добавить`.
2. Появляется модальное окно с полями ввода (`QLineEdit` и `QSpinBox`).
3. Пользователь вводит: Название: "Мастер и Маргарита", Автор: "Булгаков М.А.", Издательство: "АСТ", Томов: 1, Тираж: 5000.
4. При нажатии кнопки "Добавить", окно закрывается, контроллер формирует объект `Book` и передает его в БД.
5. Таблица на главном экране мгновенно обновляется, чтобы отобразить новую запись.

**Сценарий 3: Поиск и экспорт**
1. В верхнем меню выбирается `Записи -> Поиск`.
2. В появившемся диалоге пользователь заполняет поле "Автор" (например, "Достоевский") и указывает условие "Тираж больше 1000".
3. Нажимается кнопка "Поиск". Открывается новое окно (или обновляется текущая таблица), где показаны только записи, удовлетворяющие SQL-запросу `WHERE author = ? AND print_run > ?`.
4. Для экспорта результатов или всей базы пользователь нажимает `Файл -> Экспорт XML`. Открывается системный диалог сохранения файла, после чего DOM-парсер генерирует файл `library.xml` со строгой структурой элементов.
